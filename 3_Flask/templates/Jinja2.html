Условия в Jinja

Инструкция if в Jinja имитирует выражение if в Python. Значение условия определяет набор инструкций. Если не передать переменную или присвоить значение типа, отличного от используемого в условии, — будет получено Internal Error. В отличие от обычного if, здесь присутствует обязательное наличие закрытия условия: { % endif % }. Также можно использовать условия elif и else, как в обычном коде Python. Управляющие инструкции также могут быть вложенными

{% if age//2 > 5 %} - условие

True

- если верно {%endif%} - конец условия, обязательно

Иногда удобно записывать инструкцию if в одну строку. Jinja поддерживает такой тип записи, но называет это выражением if, потому что оно записывается с помощью двойных фигурных скобок { { … } }, а не { % … % } Else использовать необязательно. Если его нет, тогда такой результат вернет объект undefined, то есть ничего. Можно использовать сложные условия, применяя ключевые слова and и or

{{'Age больше 5' if age>5 else 'Age меньше 5'}}
